API Reference

Complete reference for the PDF Service API endpoints

GET
/health
Check service health status

Description

Returns the current health status of the PDF service, including uptime, version information, and system status.

Response Codes

200 OK - Service is healthy

500 Internal Server Error - Service is unhealthy

Response Example

JSON Response
{ "status": "healthy", "timestamp": "2024-01-15T10:30:00Z", "uptime": 3600, "version": "1.0.0", "services": { "database": "connected", "pdf_engine": "ready" } }
POST
/api/v1/bundle/bundleHtml2PDF
Convert HTML or document bundles to PDF

Description

The most powerful endpoint for PDF generation. Supports both simple HTML content and complex document bundles with headers, footers, custom styling, and advanced PDF options.

Request Parameters

Parameter Type Required Description
html string Optional Direct HTML content to convert
htmlContent string Optional Alternative field for HTML content
pdfDocumentBundle object Optional Complex document structure with head, body, header, footer
pdfOptions object Optional PDF generation options (format, margins, orientation, etc.)
emailTo string Optional Email address for notifications

PDF Options Schema

Option Type Default Description
format string "A4" Paper format: "A4", "A3", "Letter", "Legal"
landscape boolean false Page orientation
margin object {"top":"1cm","right":"1cm","bottom":"1cm","left":"1cm"} Page margins
scale number 1 Scale factor (0.1 to 2)
printBackground boolean true Include background colors and images

Response Codes

200 OK - PDF generated successfully

400 Bad Request - Invalid input parameters

500 Internal Server Error - PDF generation failed

PDF Document Bundle Schema

JSON Schema
